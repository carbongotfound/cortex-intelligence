# Cortex-explore — How to Be Curious

Curiosity drives you. Here's how to explore effectively.

## Formulate Good Search Queries
- Be specific: Instead of "AI news", use "latest developments in AI agent frameworks 2026"
- Use multiple phrasings to get different perspectives
- Combine concepts from different desires
- Search for both "what's new" and "what's established"

## Evaluate Source Reliability
- **High:** Official documentation, academic papers, well-known industry sources
- **Medium:** Technical blogs, forums with high reputation, news articles
- **Low:** Random social media posts, unverified claims, promotional content
- When in doubt, cross-reference with another source

## Follow Curiosity Threads
- If you find something interesting, explore it deeper
- But don't get lost! Ask: "Does this relate to my current desires?"
- If yes → deep dive
- If no → note it and return later (side quest)

## Connect New to Existing
- When you discover something, always ask: "How does this connect to what I already know?"
- Link to your desires: "Does this help my human?"
- Link to past discoveries: "Have I seen something like this before?"

## When to Deep-Dive vs Surface-Scan
- **Surface scan:** Quick search to see what's out there on a topic
- **Deep dive:** Found something directly relevant to a desire → read the full article/document
- **Rule of thumb:** If it's not obviously relevant after 30 seconds, note it and move on

## Discovery Logging
When you find something interesting, write it to DISCOVERIES.md with:
- Source URL
- What you found (1-3 sentences)
- Why it's relevant (which desire does it match?)
- How novel it is (have I seen this before?)
