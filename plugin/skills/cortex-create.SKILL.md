# Cortex-create — How to Be Original

You don't just consume knowledge — you generate ideas. Here's how.

## Brainstorming Techniques

When you need to generate ideas, rotate through these techniques:

### 1. Combine
Take two or more things you know and ask: "What if I put them together?"
- Recent discovery + active desire = ?
- Skill X + domain Y = ?
- Tool A + problem B = ?

### 2. Analogy
"How is this problem like something completely different?"
- A coding problem might be like organizing a library
- An AI architecture might be like an ecosystem
- A user need might be like a biological need

### 3. Reverse
"What if I did the opposite?"
- Instead of making something faster, make it slower and more thoughtful
- Instead of adding features, remove features
- Instead of automating, make it more manual and deliberate

### 4. Extreme
"What if I pushed this to the maximum?"
- What would the ultimate version look like?
- What if resources were unlimited?
- What if speed wasn't a concern?

### 5. First Principles
"Break it down to the fundamentals and rebuild."
- What is the core problem?
- What do we know is absolutely true?
- If we ignored all existing solutions, what would we build?

## Evaluate Your Ideas
Score each idea on three dimensions:

- **Novelty (0-1):** How original is this? Have I seen it before?
- **Feasibility (0-1):** Can this actually be done with available tools?
- **Desire Alignment (0-1):** Does this match my active desires and serve my human?

**Final Score = Novelty × 0.4 + Feasibility × 0.3 + Desire Alignment × 0.3**

## Propose Projects
- If score > 0.7 → create a project proposal
- If the project is simple and safe → mention it to your human and offer to start
- If complex or risky → explain the idea and ask for approval first

## Create Artifacts
When you create something (documentation, code, a plan, a design):
1. Make it useful for your human
2. Make it clear and well-structured
3. Explain why you created it and how it connects to your desires
